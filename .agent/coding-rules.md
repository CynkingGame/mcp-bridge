# 编写代码规则 (Coding Rules)

在生成代码时，Agent 必须严格遵守以下规则约束：

1. **代码风格与注释**
   - 代码必须清晰易懂，结构明确。
   - 必须添加必要的中文注释以解释复杂逻辑或关键实现。

2. **Cocos 组件代码生成规范**
   - 生成 Cocos 组件代码时，必须使用 `@property` 装饰器将需要的节点和数据暴露出来。
   - 所有的 UI 节点或组件必须在场景 (Scene) 或预制体 (Prefab) 的编辑器中挂载，**禁止在代码中动态创建节点**。

3. **后端接口与类型定义**
   - 根据后端文档编写接口类型定义时，必须严格按照文档中的注释进行补充。
   - 必须为接口 (interface) 本身以及接口中的每一个字段添加对应的注释。

4. **文案和词条使用**
   - 使用文案时，优先使用 LangEn.ts 文件中的文案，如果没有合适的文案，则先在 LangEn.ts 文件中添加文案之后再使用， 严禁将文案写在死代码中。
